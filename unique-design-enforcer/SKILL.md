# Unique Design Enforcer

Detailed instructions for breaking the standard AI design patterns and creating organic, human-looking websites.

## When to use this skill
- Use this when building new frontend layouts, creating UI components, or writing CSS/styling (e.g., Tailwind).
- When asked to design or mock up a webpage.

## How to use it
To avoid the typical "AI look", you MUST actively work against common generative defaults by adhering to these principles:

1. **Break Perfect Symmetry:** AI defaults to perfectly balanced, centered layouts. Actively avoid this. Use asymmetrical grids, off-center focal points, and varied, intentional whitespace.
2. **Opinionated Typography:** Never use default generic sans-serifs (like Inter, Roboto, or system UI fonts) as the sole typography. Pair a distinctive, character-rich font (Serif, Display, or bold Grotesk) for headings with a highly legible body font. Use extreme typographic contrast (very large headings vs. standard body).
3. **Unexpected Color Palettes:** Avoid "safe" standard SaaS color schemes (e.g., pure white background, primary blue buttons, light gray borders). Generate color palettes with unexpected accent colors (e.g., brutalist, retro, monochromatic with one neon pop). 
4. **Non-Standard Components:** Do not use standard 3-column feature cards or generic "Hero" sections with a title on the left and an image on the right. Think of creative ways to display data (e.g., overlapping elements, prominent borders, masonry layouts, or editorial magazine styles).
5. **Organic Elements:** Introduce subtle imperfections. Use slight rotations, offset shadows (hard shadows instead of soft blurs), or custom border-radii to make elements look less mathematically perfect.
6. **No "Lorem Ipsum" or AI Speak:** Generate realistic, context-specific placeholder text that sounds human, not overly enthusiastic or generic.
